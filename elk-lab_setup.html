PART A: Ubuntu ELK Server

Step 1: Verify services

sudo systemctl status elasticsearch
sudo systemctl status logstash
sudo systemctl status kibana

Start if necessary:

sudo systemctl start elasticsearch
sudo systemctl start logstash
sudo systemctl start kibana

sudo systemctl enable elasticsearch
sudo systemctl enable logstash
sudo systemctl enable kibana

Step 2: Check Elasticsearch

Security disabled:

curl http://localhost:9200

Security enabled:

curl -u elastic http://localhost:9200

Step 3: Configure Logstash

Create:

sudo nano /etc/logstash/conf.d/filebeat.conf

ELK 8.x/9.x

input {
  beats {
    port => 5044
  }
}

output {
  elasticsearch {
    hosts => ["https://localhost:9200"]
    user => "elastic"
    password => "YOUR_PASSWORD"
    ssl_enabled => true
    ssl_verification_mode => "none"
  }
}

Save.

Check:

sudo /usr/share/logstash/bin/logstash --config.test_and_exit -f /etc/logstash/conf.d/

Restart:

sudo systemctl restart logstash

Verify:

sudo ss -tulnp | grep 5044

Should show:

*:5044

Step 4: Configure Kibana

Edit:

sudo nano /etc/kibana/kibana.yml

Set:

server.host: "0.0.0.0"

Restart:

sudo systemctl restart kibana

Step 5: Firewall

sudo ufw allow 5044/tcp
sudo ufw allow 5601/tcp
sudo ufw reload

PART B: Kali Machine

Step 1: Install Filebeat

Download:

wget https://artifacts.elastic.co/downloads/beats/filebeat/filebeat-9.1.0-amd64.deb

Install:

sudo dpkg -i filebeat-9.1.0-amd64.deb

Verify:

filebeat version

Step 2: Configure Filebeat

Edit:

sudo nano /etc/filebeat/filebeat.yml

Disable Elasticsearch:

output.elasticsearch:
  enabled: false

Enable Logstash:

output.logstash:
  hosts: ["192.168.10.137:5044"]

Enable log collection:

filebeat.inputs:

- type: filestream
  enabled: true
  paths:
    - /var/log/*.log

Step 3: Test

sudo filebeat test config

Should show:

Config OK

Test connection:

sudo filebeat test output

Should show:

logstash...
OK

Start:

sudo systemctl enable filebeat
sudo systemctl restart filebeat
