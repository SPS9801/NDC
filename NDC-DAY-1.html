Information Security

WHAT IS INFORMATION SECURITY?

Think of it Like This…

Imagine you have a DIARY with your personal secrets.

You want to:
✓ Keep it PRIVATE (only you can read it)
✓ Make sure nobody CHANGES what you wrote
✓ Make sure it's AVAILABLE when you want to read it

This is exactly what Information Security does
but for DIGITAL DATA (computers, phones, internet)!

Simple Definition

Information Security = Protecting digital information from bad people and accidents

The CIA Triangle (Most Important Concept!)

Think of CIA as THREE PROMISES you make about data:

        🔒 CONFIDENTIALITY
        "Only the right people can see it"
              /\
             /  \
            /    \
           /      \
          /________\
   ✅ INTEGRITY    📡 AVAILABILITY
  "Nobody changed it"  "Always accessible"

1. Confidentiality = PRIVACY

SIMPLE MEANING:
Only authorized (permitted) people can access the data.

REAL LIFE EXAMPLE:
Your WhatsApp messages should only be read by YOU
and the person you sent them to.
NOT by strangers or hackers!

MORE EXAMPLES:
✓ Your ATM PIN - only YOU should know it
✓ Your medical records - only your doctor should see them
✓ Your salary - only HR should know it

HOW WE ACHIEVE IT:
→ Passwords (keep data locked)
→ Encryption (scramble data so only right person can read)
→ Access control (only let authorized people in)

2. Integrity = ACCURACY / NO TAMPERING

SIMPLE MEANING:
Data should be CORRECT and UNCHANGED 
unless authorized person changes it.

REAL LIFE EXAMPLE:
Imagine you scored 95 marks in exam.
Someone HACKS the system and changes it to 45.
That is an INTEGRITY violation!

MORE EXAMPLES:
✓ Bank balance should not be secretly changed
✓ Medical prescription should not be altered
✓ Legal documents should not be tampered with

HOW WE ACHIEVE IT:
→ Hashing (like a fingerprint for data)
→ Digital signatures
→ Audit logs (keep track of all changes)

3. Availability = ALWAYS WORKING

SIMPLE MEANING:
The system should work WHEN YOU NEED IT.

REAL LIFE EXAMPLE:
Imagine you go to ATM at midnight for emergency cash.
The ATM says "System Down - Try Later"
That is an AVAILABILITY problem!

MORE EXAMPLES:
✓ Hospital system must work 24/7 (life or death!)
✓ Banking apps must be available always
✓ Google/Gmail should not go down

HOW WE ACHIEVE IT:
→ Backup systems (if one fails, another works)
→ Regular maintenance
→ Protection against attacks that shut systems down

Simple OWASP Examples

SQL Injection (Simple Explanation)

WHAT IS IT?
Hacker types evil commands in a website form
instead of normal text.

NORMAL PERSON types in login box:
Username: alice
Password: mypassword123

HACKER types in login box:
Username: ' OR 1=1 --
Password: anything

WHAT HAPPENS?
The evil command fools the database:
"Give access to anyone where 1=1"
(1 always equals 1, so EVERYONE gets access!)

HOW TO PREVENT:
→ Website must clean all input before using it
→ Use "parameterized queries" (a safe way to ask database)

Cross-Site Scripting - XSS (Simple Explanation)

WHAT IS IT?
Hacker puts evil code inside a website.
When you visit that page, the code runs on YOUR computer!

EXAMPLE:
You visit a forum website.
Hacker posted this in a comment:
<script>steal_your_cookies()</script>

When you view that comment, the script RUNS
and steals your login session!

HOW TO PREVENT:
→ Website should block/remove HTML tags from user input
→ Never trust what users type in forms

PART 5: VULNERABILITY, THREAT, AND RISK

Simple Definitions First

Let's use a HOUSE ANALOGY to understand:

YOUR HOUSE = Your Computer System
YOUR VALUABLES = Your Data

VULNERABILITY = A weakness
"The back door has a broken lock"

THREAT = Someone/something that can cause harm
"A burglar is walking around your neighborhood"

RISK = The danger that actually faces you
"Burglar MIGHT come through your broken back door"

RISK = THREAT + VULNERABILITY
If no broken lock → Burglar can't enter → No risk
If no burglar → Broken lock doesn't matter → No risk
BOTH need to exist for RISK to be real!

Vulnerability - Detailed

VULNERABILITY = A weakness or flaw

TYPES OF VULNERABILITIES:

1. SOFTWARE VULNERABILITY
   Example: Windows hasn't been updated for 2 years
   The old version has known security holes!
   
   Like: A house with cracked walls
   
2. WEAK PASSWORD
   Example: Password is "123456" or "password"
   Hackers can guess this in seconds!
   
   Like: Your "lock" is just a rubber band
   
3. HUMAN VULNERABILITY  
   Example: Employee clicks on fake email link
   They didn't know it was fake (not enough training)
   
   Like: A family member who opens door for strangers
   
4. NETWORK VULNERABILITY
   Example: WiFi using old WEP security (very weak)
   Anyone can crack it with free tools online!
   
   Like: A fence with big holes in it

5. PHYSICAL VULNERABILITY
   Example: Server room has no lock
   Anyone can walk in and steal the hard drive!
   
   Like: Leaving your door wide open

THREAT = Something that CAN cause harm

TYPES OF THREATS:

👤 HUMAN THREATS:

Risk - Detailed

RISK = The actual danger you face

SIMPLE FORMULA:
RISK = THREAT × VULNERABILITY

HIGH THREAT + HIGH VULNERABILITY = HIGH RISK (Danger!)
HIGH THREAT + LOW VULNERABILITY  = MEDIUM RISK
LOW THREAT  + HIGH VULNERABILITY = MEDIUM RISK  
LOW THREAT  + LOW VULNERABILITY  = LOW RISK (Safe!)

PRACTICAL EXAMPLE:

SCENARIO: Your company's customer database

THREAT: Hackers targeting your industry (HIGH)
VULNERABILITY: No password on database (HIGH)
RISK = HIGH × HIGH = CRITICAL RISK!

AFTER FIXING VULNERABILITY: Add strong password
THREAT: Still HIGH (hackers didn't go away)
VULNERABILITY: Now LOW (strong password added)
RISK = HIGH × LOW = MEDIUM RISK (much better!)

→ MEDIUM-HIGH risk

What To Do About Risk?

4 OPTIONS (4 T's):

1. TERMINATE (AVOID the risk)
   Stop doing the risky thing altogether.
   
   Example: Stop using USB drives at work
   (USB drives can bring viruses)
   
2. TREAT (REDUCE the risk)
   Add security to make it safer.
   
   Example: Install antivirus software
   (Still use USB drives but scan them first)
   
3. TRANSFER (SHARE the risk)
   Get insurance or outsource the risk.
   
   Example: Buy cyber insurance
   (If hacked, insurance company helps pay)
   
4. TOLERATE (ACCEPT the risk)
   Sometimes risk is so small, just live with it.
   
   Example: Website outage for 1 hour/year
   (Not worth spending millions to prevent it)

PART 6: QUALITY OF SERVICE (QoS)

What is QoS?

QoS = How GOOD and RELIABLE a network connection is

SIMPLE ANALOGY:
Think of your internet connection like a WATER PIPE:

BANDWIDTH = How wide the pipe is (more water = faster internet)
LATENCY   = How long water takes to travel through pipe
JITTER    = Does the water flow smoothly or in bursts?
PACKET LOSS = Does some water spill and never arrive?

The 4 Main QoS Parameters

1. Bandwidth

WHAT IS IT?
Maximum amount of data that can travel through 
the network at one time.

SIMPLE EXAMPLE:
Highway with 6 lanes = HIGH bandwidth (many cars)
Small road with 1 lane = LOW bandwidth (few cars)

MEASURED IN: Mbps (Megabits per second) or Gbps

TYPICAL VALUES:
Home internet:    10-100 Mbps
Office internet:  100-1000 Mbps
Data center:      10+ Gbps

SECURITY IMPACT:
If bandwidth is FULL → System becomes slow → DoS attack!
Hackers can flood your network and block real users.

2. Latency (Delay)

WHAT IS IT?
Time it takes for data to travel from 
source to destination.

SIMPLE EXAMPLE:
You're playing online game.
You press FIRE button.
How long before the game shows you fired?
That delay = LATENCY

MEASURED IN: milliseconds (ms)
1000 ms = 1 second

GOOD vs BAD:
Under 20ms:  EXCELLENT (online gaming, video calls)
20-100ms:    GOOD (normal browsing, email)
100-200ms:   OKAY (feeling a bit slow)
Over 200ms:  BAD (website feels sluggish)
Over 500ms:  TERRIBLE (everything is frozen!)

SECURITY IMPACT:
High latency → Security systems respond slowly
→ Threat detection is delayed
→ You get attacked but alarm rings too late!

HOW TO TEST:
Open command prompt and type:
ping google.com

Example output:
Reply from 142.250.64.46: time=12ms ← This is latency!

3. Jitter

WHAT IS IT?
How CONSISTENT is the latency?
(Does it stay stable or keep changing?)

SIMPLE EXAMPLE:
Imagine you're watching a YouTube video:
Normal: Data arrives at steady rate → Smooth video
Jitter: Data arrives in bursts → Video keeps buffering!

ANOTHER EXAMPLE:
Good connection: 10ms, 11ms, 10ms, 12ms, 10ms
(Very consistent - small jitter)

Bad connection: 10ms, 150ms, 5ms, 200ms, 8ms
(All over the place - HIGH JITTER!)

MEASURED IN: milliseconds (ms)

ACCEPTABLE LEVELS:
Under 30ms: Good (VoIP calls, video calls fine)
30-50ms:    Okay (slight voice quality issues)
Over 50ms:  Bad (calls cutting out, echo)

SECURITY IMPACT:
High jitter → VPN connections become unstable
→ Security teams can't use remote tools properly

4. Packet Loss

WHAT IS IT?
Some data packets (small chunks of data) 
get LOST before they arrive.

SIMPLE EXAMPLE:
Imagine sending a 10-page document in 10 envelopes.
If 2 envelopes get lost in the mail,
receiver only gets 8 pages.
That's 20% PACKET LOSS!

In networks, this is measured as PERCENTAGE.

GOOD vs BAD:
Under 0.1%: Excellent (barely noticeable)
0.1-1%:     Good (slight quality decrease)
1-5%:       Poor (noticeable problems)
Over 5%:    Terrible (things barely work)

HOW TO TEST WITH PING:
ping -c 100 google.com

Output will show:
"100 packets transmitted, 99 received, 1% packet loss"

SECURITY IMPACT:
Packet loss → Security logs may not be transmitted fully
→ Attack evidence gets lost
→ Intrusion detection may miss attacks

QoS and Security Connection

HOW QoS RELATES TO SECURITY:

AVAILABILITY (from CIA Triad) = QoS

When QoS is BAD → System becomes UNAVAILABLE

EXAMPLE:
🔴 DDoS ATTACK (Distributed Denial of Service):
→ Thousands of computers send fake traffic to your server
→ Your bandwidth gets FULL
→ Latency increases dramatically  
→ Packet loss shoots up
→ REAL users can't access your website
→ AVAILABILITY is DESTROYED!

This is why QoS is a SECURITY concern!

Lab 1: Install and Use Antivirus

What is Antivirus?

ANTIVIRUS = Security guard for your computer

HOW IT WORKS (SIMPLE):

STEP 1: Antivirus has a DATABASE of known viruses
        (Like a criminal database with photos)

STEP 2: Every file that enters your computer
        is SCANNED and compared to database

STEP 3: If file matches a known virus → BLOCKED!
        If file looks suspicious → WARNING!
        If file is clean → ALLOWED!

TYPES OF DETECTION:
1. Signature-based: "I recognize this criminal's face!"
2. Behavior-based: "This person is acting suspiciously!"
3. AI/Machine Learning: "Based on patterns, this looks bad!"

Installing ClamAV on Linux (Step by Step)

# Step 1: Open Terminal
# (Right click on desktop → Open Terminal)

# Step 2: Update your system first
sudo apt update
# (This refreshes the list of available software)
# Type your password when asked

# Step 3: Install ClamAV
sudo apt install clamav -y
# (The -y means "yes" to all questions automatically)

# Step 4: Wait for installation to finish
# You'll see lots of text scrolling - that's normal!

# Step 5: Update the virus database
# (ClamAV needs to download latest virus definitions)
sudo freshclam
# This downloads information about all known viruses
# May take 5-10 minutes

# Step 6: Check it's working
clamscan --version
# Should show: ClamAV 0.103.x

echo "ClamAV installed successfully!"

Scanning Your Computer

# ================================================
# BASIC SCANNING - Easy Commands
# ================================================

# Scan ONE specific file
clamscan /home/student/downloads/file.pdf
# Output: /home/student/downloads/file.pdf: OK
# OR: /home/student/downloads/file.pdf: VIRUS FOUND

# Scan a FOLDER (and all files inside)
clamscan -r /home/student/Downloads/
# -r means "recursive" = look inside all subfolders too

# Scan with more information shown
clamscan -v /home/student/
# -v means "verbose" = show all details

# Scan and save results to a file
clamscan -r /home/ --log=/home/student/scan_results.txt
# Now you can open scan_results.txt to see what was found

# Quick scan of common locations
clamscan -r /tmp/ /home/ /var/tmp/

# Show ONLY infected files (cleaner output)
clamscan -r /home/ --infected
# --infected = only show files with viruses

Understanding Scan Output

WHAT THE OUTPUT MEANS:

GOOD OUTPUT (Clean file):
/home/student/document.pdf: OK
← "OK" means NO virus found!

BAD OUTPUT (Virus found!):

Simple Scan Script for Beginners

#!/bin/bash
# my_scan.sh - Simple antivirus scan script
# How to run: bash my_scan.sh

echo "==============================="
echo "  MY ANTIVIRUS SCAN PROGRAM   "
echo "==============================="
echo ""

# Ask user what to scan
echo "What do you want to scan?"
echo "1. Just my Downloads folder"
echo "2. My entire home folder"  
echo "3. The whole computer (takes long!)"
echo ""
echo -n "Enter your choice (1, 2, or 3): "
read choice

# Set the folder based on choice
if [ "$choice" = "1" ]; then
    SCAN_FOLDER="/home/$USER/Downloads"
    echo "Scanning Downloads folder..."
    
elif [ "$choice" = "2" ]; then
    SCAN_FOLDER="/home/$USER"
    echo "Scanning home folder..."
    
elif [ "$choice" = "3" ]; then
    SCAN_FOLDER="/"
    echo "Scanning entire computer (this will take a while!)"
    
else
    echo "Invalid choice! Scanning Downloads by default."
    SCAN_FOLDER="/home/$USER/Downloads"
fi

echo ""
echo "Starting scan of: $SCAN_FOLDER"
echo "Please wait..."
echo ""

# Run the scan
clamscan -r "$SCAN_FOLDER" --infected --log="/tmp/my_scan_result.txt"

echo ""
echo "==============================="
echo "  SCAN COMPLETE!              "
echo "==============================="
echo ""
echo "Results saved to: /tmp/my_scan_result.txt"
echo ""

# Show if any viruses found
if grep -q "FOUND" /tmp/my_scan_result.txt; then
    echo "⚠️  WARNING: Viruses were found!"
    echo "Check /tmp/my_scan_result.txt for details"
else
    echo "✅ CLEAN! No viruses found."
fi

Simple QoS Measurement Script

#!/usr/bin/env python3
# qos_simple.py
# Simple QoS measurement tool for beginners
# How to run: python3 qos_simple.py

import subprocess
import re
import statistics

def measure_latency(target="8.8.8.8", count=20):
    """Measure latency by pinging a server."""
    
    print(f"\n Measuring latency to {target}...")
    print(f" Sending {count} pings, please wait...")
    
    # Run ping command
    try:
        result = subprocess.run(
            ["ping", "-c", str(count), target],
            capture_output=True,
            text=True,
            timeout=60
        )
        output = result.stdout
    except Exception as e:
        print(f" Error: {e}")
        # Use fake data for demonstration
        return simulate_latency_data()
    
    # Extract latency values from ping output
    latencies = []
    for line in output.split('\n'):
        # Look for lines like: "64 bytes from ...: icmp_seq=1 ttl=54 time=12.5 ms"
        match = re.search(r'time=(\d+\.?\d*)', line)
        if match:
            latencies.append(float(match.group(1)))
    
    if not latencies:
        print(" Could not get latency data, using simulated data")
        return simulate_latency_data()
    
    return latencies

def simulate_latency_data():
    """Return simulated latency data for demonstration."""
    import random
    # Simulate a mostly good connection with occasional spikes
    data = [random.uniform(10, 20) for _ in range(18)]
    data.append(150.0)  # One spike
    data.append(12.0)   # Back to normal
    return data

def analyze_latency(latencies):
    """Analyze and explain latency measurements."""
    
    min_lat = min(latencies)
    max_lat = max(latencies)
    avg_lat = statistics.mean(latencies)
    
    # Calculate jitter (variation in latency)
    if len(latencies) > 1:
        diffs = [abs(latencies[i] - latencies[i-1]) 
                 for i in range(1, len(latencies))]
        jitter = statistics.mean(diffs)
    else:
        jitter = 0
    
    print("\n" + "="*50)
    print("  LATENCY ANALYSIS RESULTS")
    print("="*50)
    print(f"\n  Lowest latency:  {min_lat:.1f} ms")
    print(f"  Highest latency: {max_lat:.1f} ms")
    print(f"  Average latency: {avg_lat:.1f} ms")
    print(f"  Jitter:          {jitter:.1f} ms")
    print(f"  Samples taken:   {len(latencies)}")
    
    # Simple assessment
    print("\n  ASSESSMENT:")
    
    # Latency check
    if avg_lat < 50:
        print(f"  ✅ Latency: EXCELLENT! ({avg_lat:.1f}ms)")
        print(f"     Your connection is very fast!")
    elif avg_lat < 100:
        print(f"  ✓  Latency: GOOD ({avg_lat:.1f}ms)")
        print(f"     Your connection is fine for most uses")
    elif avg_lat < 200:
        print(f"  ⚠️  Latency: OKAY ({avg_lat:.1f}ms)")
        print(f"     Might notice some slowness")
    else:
        print(f"  ❌ Latency: POOR ({avg_lat:.1f}ms)")
        print(f"     Your connection is slow!")
    
    # Jitter check
    print()
    if jitter < 10:
        print(f"  ✅ Jitter: EXCELLENT! ({jitter:.1f}ms)")
        print(f"     Connection is very stable!")
    elif jitter < 30:
        print(f"  ✓  Jitter: GOOD ({jitter:.1f}ms)")
        print(f"     Video calls will work fine")
    elif jitter < 50:
        print(f"  ⚠️  Jitter: OKAY ({jitter:.1f}ms)")
        print(f"     Video calls might have slight issues")
    else:
        print(f"  ❌ Jitter: POOR ({jitter:.1f}ms)")
        print(f"     Video calls will have problems!")
    
    return avg_lat, jitter

def measure_packet_loss(target="8.8.8.8", count=100):
    """Measure packet loss."""
    
    print(f"\n Measuring packet loss ({count} packets)...")
    print(" Please wait...")
    
    try:
        result = subprocess.run(
            ["ping", "-c", str(count), target],
            capture_output=True,
            text=True,
            timeout=120
        )
        output = result.stdout
        
        # Find packet loss percentage
        match = re.search(r'(\d+)% packet loss', output)
        if match:
            loss = float(match.group(1))
        else:
            loss = 0.5  # Default if can't parse
            
    except Exception:
        loss = 0.5  # Simulated data
    
    print("\n" + "="*50)
    print("  PACKET LOSS RESULTS")
    print("="*50)
    print(f"\n  Packets sent: {count}")
    print(f"  Packets lost: {int(loss * count / 100)}")
    print(f"  Packet loss:  {loss}%")
    
    print("\n  ASSESSMENT:")
    if loss == 0:
        print(f"  ✅ PERFECT! No packets lost!")
    elif loss < 1:
        print(f"  ✓  GOOD! Only {loss}% lost - barely noticeable")
    elif loss < 5:
        print(f"  ⚠️  OKAY - {loss}% loss - some quality issues")
    else:
        print(f"  ❌ POOR! {loss}% loss - serious network problems!")
    
    return loss

def show_qos_summary(avg_latency, jitter, packet_loss):
    """Show final QoS summary with score."""
    
    print("\n" + "="*50)
    print("  OVERALL QoS REPORT CARD")
    print("="*50)
    
    # Calculate simple score
    score = 100
    
    if avg_latency > 50:
        score -= 20
    if avg_latency > 100:
        score -= 20
    
    if jitter > 30:
        score -= 15
    if jitter > 50:
        score -= 15
    
    if packet_loss > 1:
        score -= 20
    if packet_loss > 5:
        score -= 20
    
    print(f"\n  Latency Score:     {'✅' if avg_latency < 100 else '❌'}")
    print(f"  Jitter Score:      {'✅' if jitter < 30 else '❌'}")
    print(f"  Packet Loss Score: {'✅' if packet_loss < 1 else '❌'}")
    
    print(f"\n  OVERALL QoS SCORE: {score}/100")
    
    if score >= 90:
        grade = "A - EXCELLENT"
        advice = "Your network is in great shape!"
    elif score >= 80:
        grade = "B - GOOD"
        advice = "Minor issues, but mostly fine"
    elif score >= 70:
        grade = "C - ACCEPTABLE"
        advice = "Some problems, worth investigating"
    elif score >= 60:
        grade = "D - POOR"
        advice = "Network has issues affecting users"
    else:
        grade = "F - CRITICAL"
        advice = "Network needs urgent attention!"
    
    print(f"  GRADE: {grade}")
    print(f"  ADVICE: {advice}")
    print("\n" + "="*50)

   Risk = DANGER + WEAKNESS together
   
   Risk = Probability × Impact

5. QoS (Quality of Service)

   4 things to measure:
   • Bandwidth  = How much data fits (like road width)
   • Latency    = How fast data travels (under 100ms = good)
   • Jitter     = How consistent speed is (under 30ms = good)
   • Packet Loss = How much data gets lost (under 1% = good)
   
   Poor QoS = Availability problem = Security issue!

Common Exam Questions with Simple Answers

Q1: What is Information Security?
A: Protecting digital data from unauthorized access,
   modification, or unavailability using CIA principles.

Q2: Explain CIA Triad with examples.
A: 
   C = Confidentiality = Only right people see data
   Example: Only you can see your WhatsApp messages
   
   I = Integrity = Data is not changed without permission
   Example: Your bank balance should not be secretly changed
   
   A = Availability = System works when needed
   Example: ATM should work 24/7

Q3: What is difference between Vulnerability and Threat?
A:
   Vulnerability = A WEAKNESS (e.g., weak password)
   Threat = Something that CAN cause harm (e.g., hacker)
   Risk = When threat meets vulnerability (hacker + weak password)

Q4: What are QoS parameters?
A:
   1. Bandwidth - Network capacity
   2. Latency - Time for data to travel
   3. Jitter - Variation in latency
   4. Packet Loss - % of data lost in transit

Q5: Name OWASP Top 3 vulnerabilities.
A:
   1. Broken Access Control (94% of websites)
   2. Cryptographic Failures (83%)
   3. Injection Attacks (94%)

Q6: How much does a data breach cost on average?

   India average: ₹17.9 crore
