Basic iptables Commands (Linux Firewall)

iptables is a Linux firewall utility used to allow, block, and manage network traffic.

1. Check Current Rules

sudo iptables -L

Detailed output:

sudo iptables -L -v -n
  • -L → List rules
  • -v → Verbose
  • -n → Show IPs/ports numerically

2. Allow All Traffic (Temporary)

sudo iptables -P INPUT ACCEPT
sudo iptables -P OUTPUT ACCEPT
sudo iptables -P FORWARD ACCEPT

-P = Default policy

3. Block All Incoming Traffic

sudo iptables -P INPUT DROP

4. Allow SSH (Port 22)

sudo iptables -A INPUT -p tcp --dport 22 -j ACCEPT
  • -A → Append rule
  • -p tcp → TCP protocol
  • --dport 22 → Destination port 22
  • -j ACCEPT → Allow traffic

5. Allow HTTP and HTTPS

HTTP:

sudo iptables -A INPUT -p tcp --dport 80 -j ACCEPT

HTTPS:

sudo iptables -A INPUT -p tcp --dport 443 -j ACCEPT

6. Block an IP Address

sudo iptables -A INPUT -s 192.168.1.10 -j DROP
  • -s → Source IP

7. Allow Traffic from Specific IP

sudo iptables -A INPUT -s 192.168.1.20 -j ACCEPT

8. Delete a Rule

First list rules with numbers:

sudo iptables -L --line-numbers

Delete rule:

sudo iptables -D INPUT 1

Deletes rule number 1.

9. Flush All Rules

sudo iptables -F

-F = Flush

10. Save Rules

Ubuntu/Debian

Install:

sudo apt install iptables-persistent

Save:

sudo netfilter-persistent save

RHEL/CentOS

sudo service iptables save

11. Allow Established Connections (Stateful Firewall)

sudo iptables -A INPUT -m state --state ESTABLISHED,RELATED -j ACCEPT

This is important for SPI (Stateful Packet Inspection) firewall.

12. Allow Loopback Traffic

sudo iptables -A INPUT -i lo -j ACCEPT

lo = localhost interface

13. Basic Secure Firewall Example

sudo iptables -F

sudo iptables -P INPUT DROP
sudo iptables -P FORWARD DROP
sudo iptables -P OUTPUT ACCEPT

sudo iptables -A INPUT -i lo -j ACCEPT

sudo iptables -A INPUT -m state --state ESTABLISHED,RELATED -j ACCEPT

sudo iptables -A INPUT -p tcp --dport 22 -j ACCEPT
sudo iptables -A INPUT -p tcp --dport 80 -j ACCEPT
sudo iptables -A INPUT -p tcp --dport 443 -j ACCEPT

This:

  • Blocks unwanted traffic
  • Allows SSH
  • Allows web traffic
  • Allows established connections

Important Chains

Chain Purpose
INPUT Incoming traffic
OUTPUT Outgoing traffic
FORWARD Routed traffic

Common Targets

Target Meaning
ACCEPT Allow
DROP Silently block
REJECT Block with response

Useful Practice Tasks

  1. Allow only SSH traffic.
  2. Block one IP address.
  3. Allow ping requests.
  4. Create rules for web server.
  5. Make default INPUT policy DROP.
  6. Save and restore rules.

Check Rules Continuously

watch -n 1 sudo iptables -L -v -n

Updates every second.

Modern Alternative

New Linux systems use nftables or firewalld, but iptables is still very important in:

  • Cybersecurity
  • Linux administration
  • Networking labs
  • CEH/PenTesting
  • HPC clusters
  • Cloud VMs
